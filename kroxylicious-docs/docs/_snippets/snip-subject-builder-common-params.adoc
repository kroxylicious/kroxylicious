where:

* `<field name>` identifies the field within the certificate to be extracted.  Supported fields are listed below.
* `<match replace flags 1>`...`<match replace flags n>` series of replace-matcher used to match and map the field value into its replacement value.
* `else` defines the behavior if none of the replace-matcher match the field value.
`identity` generates a principal based on the (unmapped) field value.
`anonymous` generated an anonymous principal.
* `<principal factory>` the name of a `PrincipalFactory` implementation. Currently `UserFactory` is the only `PrincipalFactory` supported value.

Field values are tested against the replace-matchers in the order they appear in the configuration.
The first replace-matcher to match the field value generates the principal.
After a successful match, the remaining replace-matchers for this field are ignored.

If no replace-matcher matches, the outcome is defined by the `else` clause.
If there is no match and no `else` cause, no principal is generated for this field.

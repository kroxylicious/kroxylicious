= SASL Termination filter

[role="_abstract"]
The SASL Termination filter terminates SASL authentication at the proxy, authenticating clients against pluggable credential stores.
This enables centralised credential management without requiring broker-side credential configuration.

== Configuring SASL Termination

[role="_abstract"]
This procedure describes how to set up the SASL Termination filter by configuring it in Kroxylicious.

.Prerequisites

* An instance of Kroxylicious
ifdef::OpenShiftOnly[]
For information on deploying Kroxylicious, see the {OperatorGuide}.
endif::OpenShiftOnly[]
ifndef::OpenShiftOnly[]
For information on deploying Kroxylicious, see the {ProxyGuide} or {OperatorGuide}.
endif::OpenShiftOnly[]
* TLS configured for client connections to the proxy (required for security)
* Credential keystore prepared (see <<proc-managing-credentials-keystore-tool-{context}>>)

.Procedure

. Prepare credential keystore.
+
See <<proc-managing-credentials-keystore-tool-{context}>> for detailed procedures on creating keystores and managing user credentials.

. Configure a `SaslTermination` type filter.

* In a standalone proxy deployment. See <<con-sasl-termination-example-proxy-config-{context}>>
endif::[]
ifdef::include-platform-kubernetes[]
* In a Kubernetes deployment using a `KafkaProtocolFilter` resource. See <<con-sasl-termination-example-kafkaprotocolfilter-resource-{context}>>
endif::[]

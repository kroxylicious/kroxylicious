= KeystoreScramCredentialStore configuration reference

[role="_abstract"]
Configuration parameters for the KeystoreScramCredentialStore credential store implementation.

== Configuration structure

[source,yaml]
----
credentialStore: KeystoreScramCredentialStore
credentialStoreConfig:
  file: <path-to-keystore>
  storePassword: <password-provider>
  keyPassword: <password-provider>  # Optional
  storeType: <keystore-type>  # Optional
----

== Configuration parameters

[cols="2,1,3,2",options="header"]
|===
|Parameter
|Type
|Description
|Default

|`file`
|String
|Path to the Java keystore file containing SCRAM credentials. Can be absolute or relative to the proxy working directory.
|Required (no default)

|`storePassword`
|PasswordProvider
|Password to unlock the keystore. See PasswordProvider configuration below.
|Required (no default)

|`keyPassword`
|PasswordProvider
|Password to decrypt individual key entries in the keystore. If omitted, uses the same password as `storePassword`.
|Optional (defaults to `storePassword`)

|`storeType`
|String
|Java keystore type. Supported values: `PKCS12` (recommended), `JKS`. If omitted, uses the Java platform default.
|Optional (platform default)
|===

== PasswordProvider configuration

PasswordProvider allows secure password configuration without inline plaintext passwords.

.File-based password (recommended)
[source,yaml]
----
storePassword:
  passwordFile: /path/to/password-file.txt
----

The file must contain the password as plain text. Protect the file with restrictive permissions (`chmod 600`).

.Inline password (not recommended for production)
[source,yaml]
----
storePassword:
  password: "plaintext-password"
----

WARNING: Inline passwords appear in configuration files and may be logged or exposed. Use `passwordFile` in production environments.

== Keystore format

The keystore must contain SecretKey entries where:

* *Alias*: Username
* *Algorithm*: `AES` (PKCS12 requirement)
* *Encoded data*: JSON-serialized SCRAM credential

Use the `keystore-credential-tool` CLI to create and manage keystores in the correct format.
See xref:proc-managing-credentials-keystore-tool-{context}[] for details.

== Configuration examples

.Basic configuration with PKCS12
[source,yaml]
----
credentialStore: KeystoreScramCredentialStore
credentialStoreConfig:
  file: /etc/kroxylicious/credentials.p12
  storePassword:
    passwordFile: /etc/kroxylicious/keystore-password.txt
  storeType: PKCS12
----

.Configuration with separate key password
[source,yaml]
----
credentialStore: KeystoreScramCredentialStore
credentialStoreConfig:
  file: /etc/kroxylicious/credentials.p12
  storePassword:
    passwordFile: /etc/kroxylicious/store-password.txt
  keyPassword:
    passwordFile: /etc/kroxylicious/key-password.txt
  storeType: PKCS12
----

.Configuration with JKS keystore
[source,yaml]
----
credentialStore: KeystoreScramCredentialStore
credentialStoreConfig:
  file: /etc/kroxylicious/credentials.jks
  storePassword:
    passwordFile: /etc/kroxylicious/keystore-password.txt
  storeType: JKS
----

NOTE: PKCS12 is the modern, recommended keystore format. JKS is a legacy format supported for compatibility.

== File permissions

IMPORTANT: Keystore files and password files must be protected with restrictive file system permissions:

[source,bash]
----
chmod 600 /etc/kroxylicious/credentials.p12
chmod 600 /etc/kroxylicious/keystore-password.txt
chown proxy-user:proxy-group /etc/kroxylicious/credentials.p12
chown proxy-user:proxy-group /etc/kroxylicious/keystore-password.txt
----

Only the proxy process user should have read access to these files.

= Example proxy configuration file

[role="_abstract"]
If your instance of the Kroxylicious Proxy runs directly on an operating system, provide the filter configuration in the `filterDefinitions` list of your proxy configuration.

Here's a complete example of a `filterDefinitions` entry configured for SASL termination:

.Example SASL Termination configuration
[source,yaml,subs="+quotes"]
----
filterDefinitions:
  - name: my-sasl-termination-filter  <1>
    type: SaslTermination  <2>
    config:
      mechanisms:  <3>
        SCRAM-SHA-256:  <4>
          credentialStore: KeystoreScramCredentialStore  <5>
          credentialStoreConfig:  <6>
            file: /etc/kroxylicious/credentials.p12  <7>
            storePassword:  <8>
              passwordFile: /etc/kroxylicious/keystore-password.txt
            storeType: PKCS12  <9>
----
<1> Unique name for this filter instance
<2> Filter type: must be `SaslTermination`
<3> Map of SASL mechanisms to their configuration
<4> SASL mechanism name (`SCRAM-SHA-256` or `SCRAM-SHA-512` are supported)
<5> Credential store implementation (`KeystoreScramCredentialStore` for SCRAM credentials in Java KeyStore)
<6> Configuration for the credential store
<7> Path to the credential keystore file (must be readable by proxy process)
<8> Password for the keystore, provided via PasswordProvider (here using `passwordFile`)
<9> Keystore type (`PKCS12` recommended, `JKS` also supported)

where:

* `mechanisms` defines the SASL mechanisms supported by this filter. Each mechanism maps to credential store configuration.
* `credentialStore` names the credential store plugin to use. `KeystoreScramCredentialStore` loads SCRAM credentials from a Java KeyStore.
* `credentialStoreConfig` provides configuration specific to the credential store implementation.
* `file` specifies the path to the keystore file. Use absolute paths or paths relative to the proxy working directory.
* `storePassword` provides the keystore password via a PasswordProvider. Options include:
  ** `passwordFile`: read password from a file (recommended)
  ** `password`: inline password (not recommended for production)
* `storeType` specifies the KeyStore type. Use `PKCS12` (modern, recommended) or `JKS` (legacy).

.Example with SCRAM-SHA-512
[source,yaml,subs="+quotes"]
----
filterDefinitions:
  - name: my-sasl-termination-filter
    type: SaslTermination
    config:
      mechanisms:
        SCRAM-SHA-512:  <1>
          credentialStore: KeystoreScramCredentialStore
          credentialStoreConfig:
            file: /etc/kroxylicious/credentials-sha512.p12
            storePassword:
              passwordFile: /etc/kroxylicious/keystore-password.txt
            storeType: PKCS12
----
<1> SCRAM-SHA-512 mechanism provides stronger hashing than SHA-256

.Example with separate key password
[source,yaml,subs="+quotes"]
----
filterDefinitions:
  - name: my-sasl-termination-filter
    type: SaslTermination
    config:
      mechanisms:
        SCRAM-SHA-256:
          credentialStore: KeystoreScramCredentialStore
          credentialStoreConfig:
            file: /etc/kroxylicious/credentials.p12
            storePassword:
              passwordFile: /etc/kroxylicious/store-password.txt
            keyPassword:  <1>
              passwordFile: /etc/kroxylicious/key-password.txt
            storeType: PKCS12
----
<1> Optional key password (defaults to storePassword if omitted)

== Security requirements

IMPORTANT: SASL Termination requires TLS for client-to-proxy connections. Configure TLS in your virtual cluster definition to protect credentials in transit.

Example virtual cluster with TLS:

.Example virtual cluster with TLS configuration
[source,yaml,subs="+quotes"]
----
virtualClusters:
  demo:
    targetCluster:
      bootstrap_servers: kafka:9092
    clusterNetworkAddressConfigProvider:
      type: PortPerBrokerClusterNetworkAddressConfigProvider
      config:
        bootstrapAddress: proxy:9192
        brokerAddressPattern: proxy
        brokerStartPort: 9193
    tls:  <1>
      key:
        storeFile: /etc/kroxylicious/proxy-keystore.p12
        storePassword:
          passwordFile: /etc/kroxylicious/proxy-keystore-password.txt
        storeType: PKCS12
----
<1> TLS configuration required when using SASL Termination

Refer to the {ProxyGuide} for more information about configuring the proxy and TLS.

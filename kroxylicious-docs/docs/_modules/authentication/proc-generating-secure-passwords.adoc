= Generating secure passwords

[role="_abstract"]
Generate strong, random passwords for keystore passwords and user credentials following security best practices.

.Prerequisites

* Linux or Unix-like operating system with standard utilities
* OR a password manager application

== Using OpenSSL

. Generate a random password using OpenSSL:
+
[source,bash,subs="+quotes"]
----
openssl rand -base64 18  <1>
----
<1> Generates a 24-character base64-encoded random password
+
Example output:
+
[source,text]
----
kX9mP2nQ7vR4sT8uY1wZ3a
----

. For longer passwords (32 characters):
+
[source,bash,subs="+quotes"]
----
openssl rand -base64 24
----

== Using /dev/urandom

. Generate a password from /dev/urandom:
+
[source,bash,subs="+quotes"]
----
LC_ALL=C tr -dc 'A-Za-z0-9!@#$%^&*()_+-=' < /dev/urandom | head -c 20
echo
----
+
This generates a 20-character password using alphanumeric and special characters.

== Using pwgen

. If `pwgen` is installed, generate a secure password:
+
[source,bash,subs="+quotes"]
----
pwgen -s 20 1  <1>
----
<1> Generates one secure 20-character password

== Using a password manager

Password managers like 1Password, LastPass, Bitwarden, or KeePassXC can generate secure random passwords.

. Configure the password generator with:
+
* Length: at least 20 characters
* Character types: uppercase, lowercase, digits, symbols
* Avoid ambiguous characters: optional (improves readability)

. Save the generated password in the password manager for future reference

== Password requirements

Passwords must meet these minimum requirements:

* *Minimum length*: 12 characters (enforced by keystore-credential-tool)
* *Recommended length*: 20 or more characters for keystore passwords
* *Randomness*: Generated using cryptographically secure random sources
* *Complexity*: Mix of character types (not strictly required but recommended)

IMPORTANT: Following NIST SP 800-63B guidance, password length is more important than complexity rules. A 20-character random password provides strong security even without special characters.

== Storing passwords securely

*For keystore passwords:*

. Store in a file with restrictive permissions:
+
[source,bash,subs="+quotes"]
----
echo "your-generated-password" > /etc/kroxylicious/keystore-password.txt
chmod 600 /etc/kroxylicious/keystore-password.txt
chown proxy-user:proxy-group /etc/kroxylicious/keystore-password.txt
----

. Reference in proxy configuration:
+
[source,yaml,subs="+quotes"]
----
storePassword:
  passwordFile: /etc/kroxylicious/keystore-password.txt
----

*For user passwords:*

* Generate during user creation via keystore-credential-tool interactive prompts
* Never store user plaintext passwords
* Only the derived SCRAM credentials (salt, iterations, keys) are stored in the keystore

*On Kubernetes:*

. Create a Secret for the keystore password:
+
[source,bash,subs="+quotes"]
----
kubectl create secret generic sasl-keystore-password \
  --from-literal=keystore-password.txt="your-generated-password" \
  --namespace kroxylicious
----

WARNING: Never commit passwords to version control systems. Use secret management tools (HashiCorp Vault, sealed-secrets, SOPS) for production deployments.

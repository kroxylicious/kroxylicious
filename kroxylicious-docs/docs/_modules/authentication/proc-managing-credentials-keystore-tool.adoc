= Managing credentials with keystore-credential-tool

[role="_abstract"]
The `keystore-credential-tool` CLI utility manages SCRAM credentials in Java keystores.
Use this tool to create keystores and manage user credentials for SASL Termination.

.Prerequisites

* Java 17 or later
* `keystore-credential-tool` binary (distributed with Kroxylicious)

.Creating a credential keystore

. Create a new keystore:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool create \
  -k /path/to/credentials.p12 \  <1>
  -t PKCS12  <2>
----
<1> Path where the keystore will be created
<2> Keystore type (`PKCS12` recommended, `JKS` also supported)
+
You will be prompted to enter a keystore password. The password must be at least 12 characters.
+
IMPORTANT: Use a strong, randomly generated password. See xref:proc-generating-secure-passwords-{context}[].

. Set restrictive file permissions on the keystore:
+
[source,bash,subs="+quotes"]
----
chmod 600 /path/to/credentials.p12
chown proxy-user:proxy-group /path/to/credentials.p12
----
+
This prevents unauthorised access to the credential file.

.Adding a user

. Add a user with SCRAM-SHA-256 credentials:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool add-user \
  -k /path/to/credentials.p12 \  <1>
  -u alice  <2>
----
<1> Path to existing keystore
<2> Username to add
+
You will be prompted for:
+
* Keystore password (to unlock the keystore)
* User password (minimum 12 characters)
+
The tool generates SCRAM-SHA-256 credentials (salt, iterations, stored key, server key) and stores them in the keystore.

. To add a user with SCRAM-SHA-512 credentials:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool add-user \
  -k /path/to/credentials.p12 \
  -u bob \
  -m SCRAM_SHA_512  <1>
----
<1> Mechanism: `SCRAM_SHA_256` or `SCRAM_SHA_512`

.Updating a user password

. Update an existing user's password:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool update-password \
  -k /path/to/credentials.p12 \
  -u alice
----
+
You will be prompted for the keystore password and the new user password.

. The mechanism can be changed during password update:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool update-password \
  -k /path/to/credentials.p12 \
  -u alice \
  -m SCRAM_SHA_512  <1>
----
<1> Change mechanism to SCRAM-SHA-512

.Removing a user

. Remove a user from the keystore:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool remove-user \
  -k /path/to/credentials.p12 \
  -u alice
----

.Listing users

. List all users in the keystore:
+
[source,bash,subs="+quotes"]
----
keystore-credential-tool list-users \
  -k /path/to/credentials.p12
----
+
The tool displays a sorted alphabetical list of usernames with a count.

== Security considerations

WARNING: Never use the `--unlock-insecure-options` flag in production. This flag allows passwords to be passed via command-line arguments, which exposes them in process listings and shell history.

* Always use interactive password prompts (default behaviour)
* Generate strong random passwords (see xref:proc-generating-secure-passwords-{context}[])
* Protect keystore files with restrictive permissions (`chmod 600`)
* Store keystore passwords securely (use PasswordProvider in proxy configuration)
* Rotate credentials periodically according to your security policy

== Credential iteration count

The tool uses 10000 PBKDF2 iterations by default, following security best practices.
This provides strong protection against password cracking whilst maintaining acceptable authentication performance.

Higher iteration counts can be configured if required, but this increases authentication latency.

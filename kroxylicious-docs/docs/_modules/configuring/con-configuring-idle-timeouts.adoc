= Configuring idle connection timeouts

[role="_abstract"]
The proxy can automatically disconnect idle client connections to reclaim resources.
Idle timeout configuration is completely optional and disabled by default, allowing you to opt in only when needed for your deployment.

== Configuration examples

.Example: Unauthenticated timeout only
[source,yaml]
----
network:
  proxy:
    unauthenticatedIdleTimeout: 30s
virtualClusters:
  # ...
----
where:

* `unauthenticatedIdleTimeout` - specifies the length of time unauthenticated connections may remain idle before being disconnected.

.Example: Authenticated timeout only
[source,yaml]
----
network:
  proxy:
    authenticatedIdleTimeout: 5m
virtualClusters:
  # ...
----
where:

* `authenticatedIdleTimeout` specifies the length of time authenticated connections may remain idle before being disconnected.

.Example: Both timeouts configured
[source,yaml]
----
network:
  proxy:
    unauthenticatedIdleTimeout: 30s
    authenticatedIdleTimeout: 10m
virtualClusters:
  # ...
----
where:

* `unauthenticatedIdleTimeout` specifies how long unauthenticated connections can remain idle before they are disconnected.
* `authenticatedIdleTimeout` specifies how long authenticated connections can remain idle before they are disconnected.

= Configuring idle connection timeouts

[role="_abstract"]
The proxy can automatically disconnect idle client connections to reclaim resources.
Idle timeout configuration is optional and disabled by default. Enable it only when required for your deployment.
Idle timeouts are configured under `spec.network.proxy` in the `KafkaProxy` resource and apply only to the proxy listener — they are not available for the management HTTP server.
When enabled, idle disconnects are observable via the `kroxylicious_client_to_proxy_disconnects_total` metric.

== Configuration examples

.Example: Unauthenticated timeout only
[source,yaml]
----
kind: KafkaProxy
apiVersion: kroxylicious.io/v1alpha1
metadata:
  namespace: my-proxy
  name: simple
spec:
  network:
    proxy:
      unauthenticatedIdleTimeout: 30s
----
where:

* `unauthenticatedIdleTimeout` specifies the maximum idle time allowed for unauthenticated connections before they are disconnected.

.Example `KafkaProxy` configuration with unauthenticated and authenticated idle timeouts
[source,yaml]
----
kind: KafkaProxy
apiVersion: kroxylicious.io/v1alpha1
metadata:
  namespace: my-proxy
  name: simple
spec:
  network:
    proxy:
      unauthenticatedIdleTimeout: 30s
      authenticatedIdleTimeout: 10m
----
where:

* `unauthenticatedIdleTimeout` Disconnect unauthenticated connections after 30 seconds of inactivity.
* `authenticatedIdleTimeout` Disconnect authenticated connections after 10 minutes of inactivity.
